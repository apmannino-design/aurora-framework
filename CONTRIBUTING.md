# Contributing to Aurora Framework

Thank you for your interest in contributing to the Aurora Framework!

## How to Contribute

### Types of Contributions

1. **Experimental Validation**
   - Run independent experiments
   - Verify theoretical predictions
   - Test in new physical domains

2. **Theoretical Extensions**
   - Mathematical refinements
   - New domain predictions
   - Alternative derivations

3. **Code Contributions**
   - Analysis tools
   - Visualization improvements
   - Bug fixes

4. **Documentation**
   - Clarifications
   - Examples
   - Tutorials

## Submission Guidelines

### For Experimental Results

1. **Create an Issue First**
   - Describe your experiment
   - Share preliminary results
   - Get feedback before full submission

2. **Data Requirements**
   - Raw data in standard format (CSV, HDF5, JSON)
   - Analysis code (Python, MATLAB, R)
   - Clear methodology description
   - Error analysis

### For Code Contributions

1. **Follow Python Standards**
   - PEP 8 style guide
   - Type hints where appropriate
   - Docstrings for all functions
   - Unit tests for new features

## Community Guidelines

### Be Respectful
- Welcome newcomers
- Assume good faith
- Be patient with questions

### Be Scientific
- Back claims with evidence
- Acknowledge uncertainties
- Accept criticism gracefully
- Correct mistakes openly

### Be Collaborative
- Share knowledge freely
- Help others learn
- Build on existing work
- Give credit appropriately
